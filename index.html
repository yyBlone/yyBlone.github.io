首次求职面试

发表于 2023-11-01 更新于 2023-11-03 分类于 面经

go后端小厂面试总结

时间有点久了，记不太清

面试要点

老样子，先问问简历项目
保研或许算是优势，本来对方想问408和算法来着，听到保研之后几乎全问的技术
关于基础，主要问了mysql数据库，go的特性(grpc啥的)等，简单问了协程、管程方面的理解
关于组件，重点是redis、k8s和docker，具体来说：redis中用到的数据结构，k8s组成以及对它的理解等。而gin一点没问
关于拓展，重点问了对微服务的理解

面试总结

1.背八股，这个不必多说
2.重点看看岗位工作，比如第一个厂子一直在问k8s，结束之后我看了眼招聘广告才发现他们是做云原生什么的，感觉这里是应该格外准备的
3.不要求转正+长期实习+立即到岗，个人感觉是大优势
4.总结一下，一般来说都是30min左右，简历上有的都大概率问到
5.简历应该突出求职方向，比如写了熟悉maven，面试官说这个是java那个maven吗。我感觉应该在简历上删除maven，或者是写到“其他”一栏中
6.写了一个科研相关项目，但没被问到，建议从网上找个别的替换掉

关于博客（修）

发表于 1970-01-01 更新于 2023-11-03 分类于 博客

自己对博客的定位出现了误差

读了一些帖子，很有帮助，所以在此作整理

如何用在找工作上

我认为博客在简历上是一个不错的加分项，可以体现热爱技术

当然，对提升自己的能力同样有帮助

直接把博客地址写在简历里。缺点是：

1、面试官可能没有点链接

2、点了链接，发现文章很多，没时间一篇一篇去看

3、面试官拿的的一个纸质简历，总不能指望他一个一个字母的敲地址吧

如何让博客最大程度上发挥作用？

在地址后面加一句说明，对，就是一句能介绍清楚的话。例如：总共50篇原创技术文章，其中JVM相关的10篇文章，MQ相关的10篇文章。当然，面试过程中对这部分提到的知识点也很可能着重考察

另外，也可以尝试纸质简历上博客链接用二维码替代

要对文章做好归档，比如按照领域和职位分类，这样方便面试官可以提前用碎片时间了解，获得大致的印象

甚至于，投简历之前，可以针对新公司的业务炮制几篇，保证捕捉到面试官的喜好。不过目前我还没这个水平

博客的忌讳

既然有将博客写到简历的勇气

就要有把每篇文章都当做面试项目来掌握的决心

低质量、抄袭痕迹严重的博客是不可取的

必须注意，博客是写给别人的，是名片，起到宣传的作用

写什么内容

建议
  1. 有深度的技术内容，比如独到经验与实践过程，能够体现自己的专业素养
  2. 论文阅读笔记，不宜太长，可作为
  3. 可公开的工作相关内容，以及工作中的思考总结
不建议
  1. 发表激进的个人观点
  2. 写公司内部事务、薪资等容易踩雷的地方
  3. 八股文等帮助自己记忆的整理性文字
计划
  1. 一周至少写一篇技术类文章，尽量硬核有深度，保质保量吧
  2. 技术类文章得是随笔数量的2倍吧

本站需要改进的地方

  1. 页面丑，没有逼格，一看就缺少对技术的热情
  2. 生活随笔多于技术帖子，低质量，一眼水货混子
  3. 自己写博客的时候，md中hexo的部分无法自动生成，很不方便
  4. 图床设置 Typora配置smms图床-CSDN博客
  5. 有些舍本逐末，应该是先学习，然后将自己的东西作一个记录，而不是写那种很浅显的东西。而不是把课程笔记摆上去，或者对其他人的博客
  6. 如果别人的东西已经做的很好了，自己没必要重新抄一遍，建议转载
  7. 更新的过于频繁了，屁大点事都要写上去，没活硬整。今天抄八股文都想写博客上，现在想想真是病入膏肓

随笔（一）

发表于 1970-01-01 更新于 2023-11-03 分类于 随笔

年龄增长，表达的欲望愈加强烈，但瞻前顾后的怯懦显然更胜一筹

带有情绪的文字大多逃不开矫情两个字，议论性的话题又免不了立场上的争端

开启一段有深度的且真诚的交流总是磕磕碰碰的

坦诚是美好的品格，类似于拨开洋葱，把最深处的品格摆在对方的面前，是需要勇气的

记得在政治课上学过，表达是人类的欲望，我感觉确实如此

我大概就是属于从小就话多，按我妈的话说，是个话篓子

聊天、分享乃至与人交流，都是一种向外输出的方式

我们生活在信息世界，每天都被来自360度全方位的信息所挤压、填充

想到了那道小学题，泳池不能一直加水，总是要放水的

比如某家店的饭很难吃，就立即向朋友吐槽以抒发情绪上的不满，诸如此类

但表达有大有小，有深有浅

如果始终停留在琐事、浅层的表达，就好像吃小零食一样

尽管也能饱腹，但没那么健康，时常会感到言语上的匮乏与精神上的空虚

而真诚、有价值的表达就好比作了一场全身按摩，能真正的放松

事实上，我们交友聊天，也更希望对方是一本打开的书

而不是一个湿漉漉的、拿着胶带粘紧的纸盒子，哪怕盒子上画着很浓很显眼的笑脸图案

当然，就像此前说的，表达是有风险的

套马甲、开小号、叠buff等等，都是很好的选择

当然，博客也不错

因为可以自由设置无法评论、无法点赞等等hh
